Detailed Description

区画ベースではなく，壁ベースの管理ID

uint16_t にキャストすると，全部の壁が通し番号になったIDを取得できるのが特徴 迷路内部の壁の総数 WallIndex::SIZE 個の配列を確保しておけば， 取得したIDをインデックスとして使える．そのとき， WallIndex が 迷路の内部にあるかどうか確認すること．(配列の範囲外アクセス防止) isInsideOfField() 関数により迷路の内部に位置するか確認できる． 最初から全部が通し番号のIDで保持してしまうと， 迷路の範囲外の壁を表現できなくなってしまうため， 必要に応じてIDを生成するようになっている．

[x, y] : Cell Position
z : Wall Distinction in the Cell; 0:East, 1:North
=> (x, y, z) : Wall Index
